### DEC-001
- **日期**: 2026-03-29
- **主题**: M1 实现方式 - 无数据库版本
- **背景**: M1 阶段是否需要接入真实数据库
- **备选方案**:
  1. 直接接 PostgreSQL + Alembic
  2. 使用内存 / stub 版本，不接 DB
- **决策**: 采用方案2，内存版本
- **原因**: M1 目标是建立骨架和验证 API 设计，DB 接入放到 M2
- **影响**: runs.py 的 POST /official-sim/runs 目前是 stub 实现，返回固定值
- **后续动作**: M2 引入 Alembic 和真实 DB

### DEC-002
- **日期**: 2026-03-29
- **主题**: 项目目录结构
- **背景**: official-sim-server 应该放在哪个位置
- **备选方案**:
  1. 放在仓库根目录下新建 apps/official-sim-server/
  2. 放在根目录作为独立服务
- **决策**: 采用方案1，放在 apps/official-sim-server/
- **原因**: 符合 AGENTS.md 中定义的目录结构，便于与现有仓库结构保持一致
- **影响**: 新服务独立于其他 apps/
- **后续动作**: 后续按 AGENTS.md 定义的目录结构继续扩展

### DEC-003
- **日期**: 2026-03-29
- **主题**: API 路由聚合方式
- **背景**: 如何组织 FastAPI 路由
- **备选方案**:
  1. 所有路由直接挂在 main.py
  2. 使用 APIRouter 分模块，集中在 router.py 聚合
- **决策**: 采用方案2，router.py 聚合
- **原因**: 便于扩展和维护，符合 FastAPI 最佳实践
- **影响**: 路由分散在 routes/ 目录，通过 router.py 统一挂载
- **后续动作**: 后续新增平台路由只需在 routes/ 下创建文件并在 router.py 注册
