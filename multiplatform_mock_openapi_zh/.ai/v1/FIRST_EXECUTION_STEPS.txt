V1 FIRST EXECUTION STEPS

PURPOSE

This file defines the first implementation pass for V1.
The goal of the first pass is NOT to finish the whole product.
The goal is to get the core backbone running end-to-end with the smallest possible useful scope.

The first pass must achieve:
- bootable repository structure
- bootable backend services
- bootable frontend shells
- working mock platform APIs
- working unified domain APIs
- working AI suggested reply flow
- working minimal agent console
- working minimal admin console

Nothing beyond that is required for the first pass.

==================================================
STEP 0. READ THE RULE FILES FIRST
==================================================

Before generating any code, read and follow:
- DEVELOPMENT_RULES.txt
- PROJECT_CONSTRAINTS.txt
- CODEX_TASK_PROMPT.txt

If there is any conflict:
1. DEVELOPMENT_RULES.txt wins
2. PROJECT_CONSTRAINTS.txt wins next
3. CODEX_TASK_PROMPT.txt wins next
4. Any missing detail should be solved in the simplest V1-compatible way

==================================================
IMPLEMENTATION NOTES
==================================================

1. Prefer explicit code over abstraction-heavy code.
2. Prefer stable service boundaries over clever shortcuts.
3. Keep all AI logic in ai-orchestrator only.
4. Keep all platform integration logic in providers only.
5. Keep all mock logic in mock-platform-server only.
6. Keep V1 intentionally boring and predictable.

==================================================
EXECUTION STYLE
==================================================

When implementing:
- generate code step by step
- do not skip layers
- do not create speculative features
- do not add recommendation / campaign / risk / ERP logic
- do not introduce Deep Agents
- if uncertain, choose the simplest implementation that satisfies V1
